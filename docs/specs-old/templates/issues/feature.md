# 機能実装テンプレート

仕様書の準備ができて実装に入るときに使用する Issue テンプレート。

## セクション一覧

| セクション | 義務 |
|---|---|
| 概要 | 必須 |
| 背景 | 必須 |
| 関連仕様書 | 必須 |
| 実装方針 | 必須 |
| テスト方針 | 必須 |
| 実験データ | 任意 |
| 影響範囲 | 必須 |
| 完了基準 | 必須 |
| 関連 | 任意 |

## 概要 [必須]

何をするかを簡潔に記載する。

## 背景 [必須]

なぜこの Issue が必要か。

- 解決する課題・動機
- レビュー等で発見した問題の切出し時は発見経緯もここに記載

## 関連仕様書 [必須]

実装の根拠となる仕様書パス。仕様駆動開発の起点として、必ず参照先を明示する。

## 実装方針 [必須]

具体的な実装方針・設計。仕様書の What/Why に対する How をここで定義する。

## テスト方針 [必須]

テストの方針。テストファイル冒頭にコメントとして転記する。

- テスト対象の範囲
- テスト手法（単体テスト、結合テスト等）
- 重点的に検証する観点

## 実験データ [任意]

実装判断の根拠となるデータ。パフォーマンス計測・比較検討等。

## 影響範囲 [必須]

変更が及ぶファイル・機能・他 Issue との関係。

## 完了基準 [必須]

この Issue が「完了した」と判定する基準。作業のチェックリスト（`- [ ]` 形式）。

## 関連 [任意]

関連 Issue・PR・仕様書へのリンク。
