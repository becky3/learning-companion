# ドキュメント編集スキル

## 概要

既存ドキュメントの更新・修正（doc-edit）を行うスキルを定義する。

## 背景

- 仕様変更反映や情報追加など、既存ドキュメントの更新作業は頻繁に発生する
- 既存の構造を保持しながら安全に更新するための手順をスキル化し、一貫した品質を保証する

## 制約

- 編集後のドキュメントは必ずユーザーがレビューする前提とする
- 既存の構造を保持したまま更新する（セクション見出し、リスト形式、テーブル等を壊さない）

## コマンド体系

| コマンド | 用途 |
| --- | --- |
| `/doc-edit <file-path>` | 既存ドキュメントを更新 |
| `/doc-edit <file-path> "更新理由"` | 更新理由を指定して更新 |

## 処理フロー

```mermaid
flowchart TD
    A[/doc-edit 実行] --> B[対象ファイルの存在確認]
    B --> C{ファイルあり?}
    C -->|なし| D[エラーメッセージ表示]
    C -->|あり| E[ファイルを読み込み構造を理解]
    E --> F[関連するコード変更を調査]
    F --> G[ドキュメントを更新]
    G --> H[変更差分・サマリーを表示]
```

更新理由が未指定の場合はユーザーに対話的に確認する。

## 入出力

| 項目 | 内容 |
| --- | --- |
| 入力 | ファイルパス、更新理由（任意） |
| 出力 | ファイルを更新。変更前後の差分を表示 |
