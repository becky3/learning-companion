# 仕様書作成・改善テンプレート

新機能の設計時、既存仕様の修正時に使用する Issue テンプレート。

## セクション一覧

| セクション | 義務 |
|---|---|
| 概要 | 必須 |
| 背景 | 必須 |
| 対象 | 必須 |
| 詳細内容 | 必須 |
| 実験データ | 任意 |
| 影響範囲 | 必須 |
| 完了基準 | 必須 |
| 関連 | 任意 |

## 概要 [必須]

何をするかを簡潔に記載する。

## 背景 [必須]

なぜこの Issue が必要か。

- 解決する課題・動機
- レビュー等で発見した問題の切出し時は発見経緯もここに記載

## 対象 [必須]

対象の仕様書パス。新設の場合は配置先。

## 詳細内容 [必須]

新設する内容、または既存仕様書の具体的な変更点。

## 実験データ [任意]

設計判断の根拠となるデータ。調査結果・比較検討・コスト見積もり等。

## 影響範囲 [必須]

変更が及ぶファイル・機能・他 Issue との関係。

## 完了基準 [必須]

この Issue が「完了した」と判定する基準。作業のチェックリスト（`- [ ]` 形式）。

## 関連 [任意]

関連 Issue・PR・仕様書へのリンク。
