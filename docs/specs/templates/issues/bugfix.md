# バグ修正テンプレート

不具合を発見したときに使用する Issue テンプレート。

## セクション一覧

| セクション | 義務 |
|---|---|
| 概要 | 必須 |
| 背景 | 必須 |
| 再現手順 | 必須 |
| 原因 | 任意 |
| 修正方針 | 必須 |
| テスト方針 | 任意 |
| 影響範囲 | 必須 |
| 完了基準 | 必須 |
| 関連 | 任意 |

## 概要 [必須]

何をするかを簡潔に記載する。

## 背景 [必須]

なぜこの Issue が必要か。

- 解決する課題・動機
- レビュー等で発見した問題の切出し時は発見経緯もここに記載

## 再現手順 [必須]

バグの再現方法を記載する。

- 再現手順（ステップバイステップ）
- 期待動作
- 実際の動作

## 原因 [任意]

判明している場合の根本原因。推測の場合は「推測:」と明記して記載可。

## 修正方針 [必須]

どう修正するかを記載する。

## テスト方針 [任意]

回帰テストの方針。テストファイル冒頭にコメントとして転記する。

- 再現テストの追加方針
- 既存テストへの影響

## 影響範囲 [必須]

変更が及ぶファイル・機能・他 Issue との関係。

## 完了基準 [必須]

この Issue が「完了した」と判定する基準。作業のチェックリスト（`- [ ]` 形式）。

## 関連 [任意]

関連 Issue・PR・仕様書へのリンク。
